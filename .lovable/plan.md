## Що повертаю

Стару формулу з обороту: `4–8% × оборот × 22 робочих дні`, з лейблами `4% / 6% / 8%` (без "/день").

## У чому був баг

```ts
const effectiveTurnover = Math.min(budget, 3000);
```

Через жорстку стелю `3000`, будь-який бюджет ≥ 3000 USDT давав однаковий результат (3300 → $3 960, 10 000 → $3 960). Це і є "неправильна математика".

## Виправлення

Прибираю жорстку стелю `maxDailyTurnover = 3000`. Денний оборот = реальний бюджет, без штучного обрізання:

```ts
const workingDays = 22;
const dailyTurnover = budget; // було Math.min(budget, 3000)

const minDailyProfit = dailyTurnover * 0.04;
const avgDailyProfit = dailyTurnover * 0.06;
const maxDailyProfit = dailyTurnover * 0.08;

const minProfit = Math.round(minDailyProfit * workingDays);
const avgProfit = Math.round(avgDailyProfit * workingDays);
const maxProfit = Math.round(maxDailyProfit * workingDays);
```

Тепер результат лінійно росте з бюджетом і однаковий бюджет → однаковий, різний → різний.

## Підказка про soft-cap

Залишаю показ нотатки під картками для бюджету > 5 000 USDT: «результати орієнтовні — зв'яжись із саппортом для індивідуального розрахунку». Сама математика працює без обмежень.
